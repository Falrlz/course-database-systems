# Course: Database Systems

A comprehensive course that introduces fundamental concepts of database systems, including relational (SQL) and non-relational (NoSQL) approaches.

## Overview

This course covers the principles of database systems, from conceptual design to implementation and data manipulation.

Students will learn how to design structured databases, manage data efficiently, and implement both SQL and NoSQL solutions in real-world scenarios, particularly in medical informatics contexts.

## Learning Objectives

Based on course learning outcomes (CPMK), students are expected to:

* Understand fundamental database concepts and systems
* Design database structures using ERD and normalization
* Implement relational databases using SQL
* Perform data manipulation and analysis (DDL, DML, JOIN, VIEW, etc.)
* Understand and apply NoSQL concepts using MongoDB

## Topics Covered

### 1. Introduction to Database Systems
* Data, information, and database concepts
* Traditional systems vs database systems
* Advantages of database systems (speed, accuracy, security) 

### 2. Database Components & Architecture
* DBMS, database, users, and applications
* Hardware and operating systems
* Data abstraction (physical, logical, view levels) 

### 3. Database Design (ERD)
* Entity, attribute, relationship
* Cardinality (1:1, 1:N, N:M)
* Conceptual, logical, and physical design 

### 4. Normalization
* 1NF, 2NF, 3NF, BCNF
* Functional dependency
* Redundancy and anomaly reduction 

### 5. SQL Fundamentals
* DDL (CREATE, ALTER, DROP)
* DML (INSERT, UPDATE, DELETE)
* SELECT, WHERE, JOIN

### 6. SQL Programming and Advanced Features
* Aggregate functions (SUM, COUNT, AVG, MIN, MAX) 
* VIEW (virtual table concept) 
* Stored Procedure (reusable SQL logic) 
* Trigger (event-driven automation in database) 

### 7. Database Case Study (Medical Systems)
* Patient examination system
* Clinic registration system
* SQL-based analysis and reporting 

### 8. Introduction to NoSQL (MongoDB)
* Document-oriented database
* BSON and data types
* Relational vs document model 

### 9. MongoDB Query & Aggregation
* Comparison operators ($eq, $gt, $lt, etc.)
* Logical operators ($and, $or, etc.)
* Aggregation pipeline ($group, $lookup, $project, etc.)

## Learning Path

Follow this structured learning progression:

1. Study fundamental concepts in `modules/`
2. Learn database design (ERD & normalization)
3. Practice SQL queries and database manipulation
4. Explore advanced SQL (JOIN, VIEW, TRIGGER, PROCEDURE)
5. Learn NoSQL concepts using MongoDB
6. Apply knowledge in case studies and projects
7. Complete assessments in `lab-sessions/` (Midterm Exam, Final Exam)

## Example Case Studies

This course includes real-world database scenarios such as:

* Clinic patient examination system
* Medical registration system
* Pharmacy and healthcare data management

Each case requires:

* Design ERD
* Normalize database structure
* Implement SQL queries
* Perform data analysis
